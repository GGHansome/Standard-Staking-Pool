# 产品功能清单 (Features) - V1 标准双币质押收益池

基于产品需求文档 (PRD)，以下是 V1 标准双币质押收益池 (Standard Dual-Token Staking Pool) 的详细功能清单。该清单从用户端、运营端、安全风控以及数据支持四个维度进行了梳理。

## 1. 用户端核心功能 (User Features)

*   **无锁仓随存随取 (Flexible Staking)**：用户可以随时存入 (Stake) 和提取 (Withdraw) 质押代币，没有任何锁仓期限制，保证资金的绝对流动性。
*   **双币解耦/异币挖矿 (Dual-Token Support)**：支持质押代币与奖励代币为不同的 ERC20 资产（例如质押 LP Token，赚取 USDC 或平台治理代币），同时也兼容同币复投。
*   **动态收益瓜分 (Dynamic APR)**：基于“水位线模型”，系统根据用户当前质押量占资金池总锁仓量 (TVL) 的比例，按秒动态计算并分配奖励。
*   **实时收益结算与领取 (Real-time Reward Claiming)**：用户的每一笔资金变动（存、取）都会自动前置触发收益结算。用户也可以随时单独调用领取功能 (Claim Reward) 提取已产生的收益。
*   **一键安全退出 (One-Click Exit)**：提供便捷的 `exit()` 功能，允许用户通过一次操作同时完成“全额提取本金”和“领取所有未结收益”。

## 2. 运营与管理功能 (Operator & Admin Features)

*   **双层权限分离 (Dual-Role Access Control)**：
    *   **普通管理员 (Operator)**：仅拥有注入奖励的权限，方便团队使用自动化脚本（Bot）定期打款，无法触碰资金或修改核心参数。
    *   **超级管理员 (Owner/Admin)**：必须由多签钱包控制，拥有设置周期、紧急暂停和误打资金救援等高危特权。
*   **奖励平滑展期机制 (Reward Smoothing)**：当运营团队在当前奖励周期未结束时注入新奖励，系统会自动将“剩余未发奖励”与“新注入奖励”合并，并在全新的周期内平滑释放。此功能可有效防止“巨鲸”资金的瞬间套利和三明治攻击。
*   **安全修改奖励周期 (Safe Duration Update)**：超级管理员可以修改单次奖励的发放周期时长（如 7 天），但系统严格限制该操作**只能在当前发奖周期完全结束后**执行，防止收益计算错乱。

## 3. 安全与风控功能 (Security & Risk Management)

*   **紧急暂停与降级服务 (Emergency Pause & Graceful Degradation)**：在遭遇极端情况或黑客攻击时，超级管理员可触发暂停。**核心安全底线**：暂停状态下仅阻断新资金存入，但**强制允许**用户撤出本金和提取已结算收益，保障用户的“最后撤离权”。
*   **奖励池余额强校验 (Reward Insolvency Protection)**：在管理员注入奖励时，合约会严格校验当前真实的奖励代币余额是否充足，防止因“手误少打钱”导致后期用户领奖失败（空头支票）。
*   **误打资金逃生舱 (ERC20 Rescue)**：如果用户误将不支持的代币打入合约，超级管理员可以将其提取并退还。**红线限制**：代码层面严格锁死，超级管理员绝对无法提取用户的“质押本金”以及“尚未发放的奖励代币”。
*   **严格的防重入与精度保护**：
    *   领奖逻辑严格遵循“先清零内部账本，再执行外部转账”的安全模式。
    *   收益计算采用“先乘后除”及 $10^{18}$ 精度放大，防止极小资金或极短时间导致的精度丢失和除零异常。
*   **标准代币安全交互 (SafeERC20)**：所有代币转账强制包裹 OpenZeppelin 的 `SafeERC20` 库，兼容各类非标准 ERC20 代币（注：V1 暂不支持带有通缩/交易税机制的代币）。

## 4. 数据与前端支持功能 (Data & Integration Features)

*   **实时数据视图 (View Functions)**：为前端 DApp 提供丰富的实时查询接口：
    *   `totalSupply()`: 查询资金池总锁仓量。
    *   `rewardRate()`: 查询全局奖励释放速率（用于前端计算并展示实时 APR）。
    *   `earned(address)`: 查询指定用户的实时未领收益（用于前端数字跳动效果）。
    *   `balanceOf(address)`: 查询指定用户的当前质押本金。
*   **完整的链上事件追踪 (Event Logging)**：关键业务动作完整抛出事件（`Staked`、`Withdrawn`、`RewardPaid`、`RewardAdded`），方便 The Graph 或后端索引器进行数据统计与面板分析。
